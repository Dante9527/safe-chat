# 📘 SafeChat 工地使用手冊

> **本文件為工地主任、安全衛生管理員、作業員的日常操作指南。**
> 不需要懂程式，照著步驟就能上手。

---

## 🎯 這個東西能幫我什麼？

想像一下你平常遇到的狀況：

- 🔍 新進人員問「施工架搭設的安全規定是什麼？」→ 你要翻 100 頁的手冊找答案
- 📋 工地發生意外要通報，但通報流程細節記不清楚
- ⚡ 臨時要檢查某項作業有沒有違反規定，但法規太多查不完
- 👷 外勞聽不懂台語，工地主任忙不過來講解安全規範

**SafeChat 就是工地的 AI 安全顧問**，用自然語言問問題，它會從你匯入的法規裡找答案並告訴你出處。不是 Google，不是 ChatGPT，是**專屬於你工地的知識庫**。

### 跟直接用 ChatGPT 差在哪？

| 面向 | ChatGPT | SafeChat |
|------|---------|---------|
| **資料來源** | 網路上學來的 | 你匯入的法規手冊 |
| **可信度** | 可能唬爛（幻覺） | 每個答案都標註來源頁碼 |
| **機密性** | 資料傳到雲端 | 全部在本機，資料不出場 |
| **網路需求** | 一定要連網 | 離線也能用 |
| **成本** | 按使用量付費 | 裝一次，用到爛 |

## 🚀 3 分鐘快速上手

### 情境 1：我只想問問題

1. **打開網頁** — 瀏覽器輸入 `http://工地伺服器IP:8000`
   （你的 IT 會給你這個網址）

2. **看到 SafeChat 畫面** — 左邊是文件區、中間是對話區、下面是輸入框

3. **點「快速提問」按鈕** 或**直接打字**提問：

   ```
   施工架搭設有哪些安全規範？
   ```

4. **答案會即時逐字出現** — 不需等全部生成完才顯示，字會像打字一樣蹦出來。結束後下方會出現 `📎 營造安全衛生設施標準 p.12 85%` 這樣的來源標籤

5. **點開來源** — 如果對答案有疑問，可以翻原始法規第 12 頁確認

### 情境 2：我要加新的安全手冊進來

假設公司剛發了一份新的「高空作業 SOP.pdf」：

1. **把 PDF 檔放到左側上傳區** — 直接拖進去，或點一下選檔案（支援多選）
2. **等待上傳** — 會看到即時進度「(1/5) 檔名」，每個檔案約 10-30 秒
3. **看到訊息「全部完成：N 個檔案匯入成功」** — 完成
4. **開始提問** — 新文件的內容現在就能被查詢了

## ❓ 常見問題

### Q1：AI 回答的答案不對或不完整怎麼辦？

**優先檢查「來源」。** 每個回答下方都有 `📎 文件名 p.頁碼 相關度%` 的標籤。

- 如果**相關度低於 60%**，代表 AI 沒找到很匹配的資料 — 試著換一種問法
- 如果相關度高但答案仍怪，請翻閱原文確認，可能是**手冊本身資訊不足**
- 依然不確定？**以法規原文為準，AI 只是輔助**

**永遠記得：SafeChat 是查詢工具，不取代人類專業判斷。**

### Q2：看到「系統目前以離線模式運作」是什麼意思？

代表 **AI 推論服務暫時斷線**（可能是伺服器在更新、網路有問題、或 Ollama 沒啟動）。

此時 SafeChat 會**降級運作**：
- ✅ 仍然會**檢索相關法規原文**給你看
- ❌ 但**不會合成摘要回答**
- 🔔 畫面右上角會出現**黃燈或紅燈警示**

**怎麼辦？**
- 先用顯示的法規原文應急
- 聯繫 IT 確認 AI 服務狀態
- 待恢復後重新提問即可

### Q3：我的問題涉及機密，這樣打進去安全嗎？

**部署在工地本機的 SafeChat 是安全的。** 所有資料：
- 只存在工地辦公室的這台機器上
- **不會**傳到任何雲端服務
- 斷網也能用，代表資料**根本沒有對外傳輸的路徑**

### Q4：新人問的問題 AI 答不出來，我能教它嗎？

可以，有兩種做法：

**做法 A：補充文件（最簡單）**
- 把公司內部的 SOP、Q&A 文件整理成 PDF 或 TXT
- 匯入到 SafeChat
- 新人下次問類似問題就能查到了

**做法 B：整理 FAQ 文件**
- 把常見 30 個問答寫成 `FAQ_工地常見問題.txt`
- 匯入後，AI 會從 FAQ 中找答案
- 比修改法規文件容易得多

### Q5：系統跑很慢怎麼辦？

**SafeChat 採用串流輸出，第一個字通常不到 1 秒就會出現**，之後字會持續蹦出直到回答完整。

**總生成時間**取決於硬體（預設使用 Llama 3.1 8B）：

- **M5 Mac + 32GB RAM：** 約 15-35 秒完整回答，首 token ~5-10 秒
- **M1/M2 Mac + 16GB RAM：** 約 15-25 秒完整回答，首 token ~10-15 秒
- **i5 + 16GB RAM（CPU only）：** 約 30-60 秒完整回答

**加速方法：**
1. 請 IT 換用**較小的模型**：`llama3.2:3b`（品質略降但快 3 倍）
2. 請 IT 加裝**顯示卡（GPU）**：回答速度可提升 10 倍以上
3. 改用**雲端 API**：最快但要付費且資料外傳

### Q6：要怎麼清掉測試用的文件重新匯入？

左側底部有「🗑 清空知識庫」按鈕，點一下後系統會要求輸入**管理員密碼（Admin Token）**。

1. 點「清空知識庫」
2. 按確認
3. 輸入管理員密碼（請向 IT 索取）
4. 完成

**注意：** 這個動作不可回復，請確認後再按。沒有管理員密碼無法清空。

### Q7：系統要求輸入 API Key 怎麼辦？

如果 IT 有設定 API Key 保護，第一次使用時需要在瀏覽器中設定：

1. 向 IT 索取 API Key
2. 在瀏覽器的開發者工具 Console 輸入：
   ```
   sessionStorage.setItem('safechat_api_key', '你的API Key')
   ```
3. 重新整理頁面即可

若 IT 沒有設定 API Key（封閉內網環境），則不需要此步驟。

## 💰 效益說明（給主管看的）

### 時間成本試算

**每位工地主管每天處理的詢問**：假設 5 次

| 項目 | 傳統方式 | 用 SafeChat |
|------|---------|------------|
| 單次查詢時間 | 5-10 分鐘翻手冊 | 30 秒 |
| 每天花費時間 | 25-50 分鐘 | 2.5 分鐘 |
| **每週省下時間** | — | **約 2-4 小時** |

以工務主管時薪估算，每人每月可省下 1-2 個工作天的詢問時間。

### 工安合規效益

- **減少「沒人問所以沒查」的違規風險** — 新人更願意問 AI（不怕被嫌煩）
- **降低外勞溝通障礙** — 法規中文太難懂，AI 可轉述成白話
- **留下查詢紀錄** — 未來可分析最常被問的問題，優先加強訓練

### 導入 vs. 違規成本

單一違規罰款範例：

- 施工架未依規定搭設：**3 萬 - 15 萬**
- 未設安全防護：**3 萬 - 30 萬**
- 發生職業災害：**30 萬起**，另加民事、刑事責任

**SafeChat 建置成本（一次性）：**
- 硬體（Mini PC + 16GB RAM）：約 **2-3 萬**
- 軟體：**開源免費**
- IT 部署：**1-2 天人力**

**結論：避免一次輕微違規就回本。**
